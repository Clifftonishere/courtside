# Courtside — NBA Intelligence Platform
## Replit Build Prompt

You are building **Courtside**, an AI-powered NBA intelligence platform. The tagline is **"See the edge."** It combines real-time game data, a quantitative prediction model, AI-generated editorial analysis, prediction market odds, and comprehensive player scouting — served through a clean, data-rich UI designed for basketball obsessives who think like analysts.

Courtside is NOT a betting platform. It is an educational sports analysis tool that helps fans understand games more deeply. It surfaces probabilities, prop analysis, and market comparisons — but always with a disclaimer that this is not financial advice.

---

## Tech Stack
- **Framework:** Next.js 14 (App Router)
- **Styling:** Tailwind CSS
- **Fonts:** Load from Google Fonts:
  - `JetBrains Mono` (weights: 400, 500, 600, 700) — for all data, numbers, stats, tickers, odds, percentages, monospace elements
  - `Outfit` (weights: 300, 400, 500, 600, 700) — for all body text, headings, labels, navigation, editorial content
- **Data:** Mock data hardcoded in a `/lib/mock-data.ts` file (we'll wire real APIs later)
- **Charts:** Recharts for win probability charts and stat visualizations
- **Icons:** Lucide React

## Design System

### Philosophy
Analytical precision meets insider access. Think Bloomberg Terminal meets The Athletic. Data-dense but not cluttered. Every pixel earns its space. The UI should feel like sitting courtside with a quantitative analyst whispering in your ear.

### Typography Rules
- **Headlines/Navigation:** Outfit 600-700, tight letter-spacing (-0.02em)
- **Body text:** Outfit 400, line-height 1.6
- **All numbers, stats, scores, odds, percentages:** JetBrains Mono. This is critical — every number on the platform should be monospace.
- **Labels/Captions:** Outfit 400-500, 11-12px, uppercase letter-spacing where used as section headers
- **Data labels:** JetBrains Mono 11px, uppercase, letter-spacing 0.5-1px, text-muted color

### Spacing & Layout
- Max content width: 1200px, centered
- Card border-radius: 12px
- Inner card padding: 20-24px
- Section gaps: 24-32px
- Use subtle 1px borders, not shadows
- Cards have a slight border, not box-shadow

### Confidence Tier Colors
- **HIGH:** Green (#16a34a)
- **MED:** Amber (#d97706)
- **COND (Conditional):** Gray (#9ca3af)

## Pages & Components

### 1. Global Shell
**Header** (fixed top, all pages):
- Left: `courtside` wordmark in JetBrains Mono 700, 20px. Next to it: `see the edge` in JetBrains Mono 400, 11px, muted color
- Center: Navigation pills — `tonight` | `players` | `arena` — styled as a segmented control with active state having white background + subtle shadow
- Right: Season accuracy record displayed as compact stats: `187W · 142L · W4 · 56.8%` in JetBrains Mono 11px. GREEN for wins, RED for losses.

**Ticker** (below header, horizontal scrolling):
- Auto-scrolling horizontal ticker showing: live score updates, completed game results, upcoming tipoff times, notable stat milestones from live games
- Pauses on hover
- Monospace font for all numbers
- Green dot pulse animation for live games
- Format examples: `🟢 LAL 87 - DEN 94 Q3 4:22` | `BOS @ NYK 7:30 PM` | `Jokic: 28/14/11 — triple-double watch`

### 2. Tonight Page (Main/Home — `/`)

This is the core experience. Three sections stacked vertically:

#### Section A: Ask Bar (THE centrepiece)
- Large, prominent input bar at the top of the page
- Placeholder: `Ask anything — "Will Brunson score 30 tonight?" or "Jokic triple-double odds?"`
- Right-aligned "Analyze" button inside the input, dark background (#1A1A18), white text
- Below input: row of example query chips (italic, bordered pills). Examples:
  - "Will Brunson score 30+ tonight?"
  - "Jokic triple-double vs Milwaukee?"
  - "SGA over 6 assists vs Lakers?"
  - "Curry 5+ threes tonight?"
  - "Who wins Celtics vs Knicks?"
- On submit: show a loading state ("Analyzing across 77,738 game logs, live markets, form signals...") with a shimmer animation
- Then render the **Analysis Card** (see component spec below)

#### Section B: Tonight's Games (below ask bar)
- Section header: `TONIGHT'S GAMES` in JetBrains Mono 11px uppercase, muted, with date on the right
- Vertical list of game cards, one per row
- Each game card shows:
  - Away team (badge + name + record) on left
  - Game status in center (time for upcoming, quarter+clock for live with green pulse dot, "Final" for completed)
  - Home team (badge + name + record) on right
  - Courtside pick on far right: team abbreviation in confidence color, probability percentage, predicted margin, confidence tier badge
  - For final games: show "CORRECT" (green) or "MISS" (red) next to the pick
- Clicking a game card expands it OR navigates to a game detail page showing:
  - Full Courtside Take (editorial analysis paragraph, Mamba Mentality voice — decisive, specific, never hedging)
  - Power Model breakdown (factor-by-factor: net rating, HCA, rest, form, H2H, injuries — each with a contribution bar)
  - Win probability chart (if live or final)
  - Box score (if live or final)
  - Community vote (AGREE / pick the other side — show consensus %)
  - Market odds from Kalshi (spread, moneyline, total)

#### Section C: Quick Insights (optional sidebar or bottom section)
- "Trending" section showing 3-4 notable signals:
  - "Jokic has recorded a triple-double in 4 of his last 5 games"
  - "Boston is 2-8 in their last 10 road games"
  - "Brunson averaging 31.2 pts over last 5 (season avg: 28.4)"
- Each insight clickable → pre-fills the ask bar with a related query

### 3. Players Page (`/players`)

This is the DNA tab from IRIS.

#### Player List
- Grid of player cards (3-4 columns on desktop, 2 on mobile)
- Each card shows: player name (Outfit 600), team badge, position, tier badge
- Tier badges with visual hierarchy:
  - **Generational** — Gold border + subtle glow (#FFD700)
  - **All-World** — Silver border (#C0C0C0)
  - **Franchise** — Bronze border (#CD7F32)
  - **All-Star** — Neutral gray
  - **Rising** — Light gray, no border accent
- Key season stats in JetBrains Mono: PTS / REB / AST / FG%
- Search bar at top to filter by name
- Filter pills below search: position (PG, SG, SF, PF, C), team, tier

#### Player Profile Page (`/players/[id]`)
**This is the "shrine" page — the most visually rich page in the app.**

Dark background (#0A0A0B) for this page only. All other pages use the light theme.

Sections stacked vertically:

1. **Hero Section**
   - Player name in Outfit 700, 48px, white
   - Team + Position + Jersey Number
   - Core season stats in large JetBrains Mono: PTS / REB / AST / FG%
   - Attribute radar chart (hexagon) showing 6 categories: Inside Scoring, Outside Scoring, Playmaking, Athleticism, Defending, Rebounding (0-99 scale)
   - Tier badge prominently displayed

2. **Courtside Identity** (one defining sentence)
   - Example: "The most unguardable scorer in basketball — full stop."
   - Outfit 500, 20px, white/85 opacity
   - Character-by-character streaming animation on load

3. **Tonight's Props** (ONLY shows if player has a game today — this is the Edge integration)
   - Header: `COURTSIDE EDGE — TONIGHT'S ANALYSIS` in JetBrains Mono
   - For each stat (points, rebounds, assists):
     - Stat name + threshold + probability + confidence tier
     - Probability bar (colored by over/under lean)
     - Key context: L5 avg vs season avg vs opponent
     - Opponent defensive rating for that stat
   - Market comparison if available (FanDuel/DraftKings line vs Courtside model)
   - Edge signal callout if model diverges from market by >5%

4. **Dissection** (4 glass-morphism cards in 2x2 grid)
   - Each card: trait category (OFFENSE/DEFENSE/PLAYMAKING/PHYSICALITY), skill label, editorial detail
   - Below: weakness cards with red accent border and "EXPLOITABLE" tag

5. **Scout Notes** (3-4 fictional scout quotes)
   - Styled as blockquotes with attribution
   - Written as if real basketball insiders are talking

6. **Scouting Report** (2-3 paragraph narrative)
   - What does watching this player feel like?

7. **Last 5 Games** (compact table)
   - Opponent, W/L, PTS, REB, AST, +/-
   - Color-coded results

8. **Tags** (horizontal pill row)
   - Examples: ASCENDING, CLUTCH, UNDERRATED, LOCKED-IN, DECLINING

### 4. Arena Page (`/arena`)

Editorial content hub. Blog-style layout.

- Modern card-based layout with featured article hero at top
- Content types: Pre-Game Previews, Post-Game Verdicts, Player Spotlights, Weekly Power Rankings
- Each article card shows: headline, 2-line excerpt, date, reading time, category tag
- Newsletter signup CTA at bottom of page

## Important Behavioral Notes

1. **The Ask Bar is the hero interaction.** Every other feature supports it. Users come to Courtside to ask a question and get a quantified answer.

2. **Every number is monospace.** Scores, odds, percentages, stats, records — all in JetBrains Mono. This is non-negotiable for the analytical precision feel.

3. **Confidence tiers are color-coded everywhere.** HIGH = green, MED = amber, COND = gray. These colors appear on game cards, analysis results, and pick badges.

4. **The accuracy record is always visible** in the header. This is Courtside's credibility — a running W-L record that proves the model works (or doesn't). Transparency is the brand.

5. **Dark theme ONLY for player profile pages.** Everything else uses the light warm theme. The player profile ("shrine") pages use a near-black background (#0A0A0B) for cinematic impact.

6. **"Educational analysis, not financial advice"** appears at the bottom of every analysis card and in the site footer. This is a legal necessity.

7. **The editorial voice is Mamba Mentality** — decisive, specific, never hedging. Courtside doesn't say "OKC should probably win." It says "SGA drops 35 on a Dallas defense with no answer. HIGH confidence." Every take names a player, calls a shot, predicts a moment.

8. **Mobile-first responsive design.** The ask bar, game cards, and analysis cards should all work beautifully on phone screens. The player profile page should stack vertically on mobile.
